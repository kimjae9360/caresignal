# 케어시그널 (CareSignal) 프로젝트 구조 분석 보고서

이 보고서는 v0로 생성된 **케어시그널 (CareSignal)** 웹 애플리케이션의 폴더 구조, 핵심 파일의 기능, 프론트엔드와 백엔드의 역할 분담, 그리고 해커톤을 위해 향후 Gemini API를 연동하여 고도화할 방향을 설명합니다.

---

## 1. 프로젝트 개요 (Overview)

* **프로젝트명**: 케어시그널 (CareSignal)
* **주요 타겟**: 돌봄종사자 (사회복지사, 요양보호사 등)
* **핵심 기능**: 어르신의 스마트 기기(웨어러블 밴드, 스마트 약통) 센서 데이터와 현장 방문 음성 기록을 AI가 분석하여, 복지사에게 위험도가 높은 어르신을 순서대로 추천하고(트리아지), 방문 대응 행동 지침 및 보호자/의료진 공유용 전문 보고서를 자동 생성합니다.
* **기술 스택**:
  * **프레임워크**: Next.js 16.2 (App Router 기반)
  * **언어**: TypeScript
  * **스타일링**: Tailwind CSS
  * **아이콘**: Lucide React
  * **컴포넌트 라이브러리**: shadcn/ui (`components/ui/button.tsx`)

## 3. 핵심 파일별 기능 설명 (Key Files)

### 1) [page.tsx](file:///c:/care/app/page.tsx)
* **기능**: 애플리케이션의 화면 이동(라우팅) 및 전역 상태를 중재하는 **컨트롤러** 역할을 수행합니다.
* **주요 상태(State)**:
  * `view`: 현재 보고 있는 대화면 (`"auth"`: 로그인, `"dashboard"`: 메인 대시보드, `"detail"`: 어르신 상세 분석).
  * `section`: 사이드바에서 선택한 세부 탭 (`"dashboard"`: 종합, `"safety"`: 안전 관리, `"sensors"`: 센서 현황, `"voice"`: 음성 기록, `"settings"`: 설정).
  * `selectedId`: 현재 상세보기를 하고 있는 어르신의 고유 ID.
* **흐름**:
  * 최초 접속 시 `AuthScreen`을 띄웁니다.
  * 로그인을 성공하면 `Dashboard` 화면이 로드됩니다.
  * 대시보드 리스트나 알림판에서 특정 어르신을 클릭하면 `selectedId`를 업데이트하고 `PatientDetail`로 진입합니다.

### 2) [auth-screen.tsx](file:///c:/care/components/auth-screen.tsx)
* **기능**: 사회복지사 전용 로그인/회원가입 화면입니다.
* **주요 UI**:
  * 왼쪽 영역: 서비스 소개 (스마트 기기 실시간 연동, 음성 AI 자동 분석, 다제약물 위험 트리아지 안내).
  * 오른쪽 영역: 아이디/비밀번호 입력 폼 및 해커톤 데모를 위한 **"데모 로그인" 우회 버튼**을 제공합니다.

### 3) [dashboard.tsx](file:///c:/care/components/dashboard.tsx)
* **기능**: 메인 화면으로, 사이드바 탭 전환에 따라 5가지 세부 세션을 전환하여 렌더링합니다.
* **핵심 컴포넌트**:
  * `TriagePatientList`: AI가 매긴 위험도 점수(`riskScore`)를 기준으로 어르신을 내림차순 정렬하여 긴급, 주의, 안정 상태를 표 형태로 일목요연하게 표시합니다.
  * `SensorsSection`: 각 어르신별 스마트 약통과 웨어러블 밴드가 현재 연결되어 있는지 직관적으로 알려줍니다.
  * `VoiceSection`: 어르신 가정에 방문하여 요양 기록이나 대화를 작성/녹취한 음성 기록 요약을 리스트 형태로 제공합니다.
  * `SettingsSection`: 모니터링 알림 조건, SMS 자동 발송 등 환경 설정을 조정할 수 있습니다.

### 4) [patient-detail.tsx](file:///c:/care/components/patient-detail.tsx)
* **기능**: 특정 어르신의 신체 및 환경 데이터를 정밀하게 심층 분석하는 핵심 페이지입니다.
* **화면 구성**:
  * **수면 시간 추이**: 최근 7일간의 수면 시간을 막대 그래프로 보여주어 수면 시간 이상 저하(예: 수면 부족 3.5시간)를 빠르게 포착하게 합니다.
  * **실시간 복약 현황**: 오늘 아침, 점심, 저녁 약 복용 일정을 보여주고 스마트 약달력과 연동하여 복약 여부(`done` 여부)를 표시합니다.
  * **현장 음성 기록 & AI 인과 매핑**: 복지사의 음성 녹음 기록과 이로부터 AI가 결론 내린 의심 증상, 분석 신뢰도를 시각적으로 매핑하여 보여줍니다.
  * **행동 지침 및 공유 리포트**: 복지사 맞춤 방문 지침 가이드를 제안하고, 의료진/보호자에게 전송할 수 있는 소견서를 생성하여 카카오톡(알림장) 전송 모의 동작을 수행합니다.

### 5) [data.ts](file:///c:/care/lib/data.ts)
* **기능**: 현재 애플리케이션의 핵심 모델(인터페이스) 타입 정의와 목(Mock) 데이터를 보관하는 데이터 레이어 파일입니다.
* **어르신 가상 데이터**:
  1. **김순자 어르신 (긴급/위험도 88점)**: 새로 추가된 관절염약 복용 후 어지러움 발생 및 최근 3일간 극심한 수면 저하를 나타내고 있음 (다제약물 상호작용 의심 사례).
  2. **이영수 어르신 (주의/위험도 64점)**: 혈압약 복용 시간이 지연되고 있으며, 웨어러블 분석 결과 산책/외출 등 활동량이 감소 중임.
  3. **박말순 어르신 (안정/위험도 22점)**: 약 복용과 수면 패턴이 매우 양호하며 평온한 상태를 유지 중임.

## 4. 프론트엔드 vs 백엔드 구조 분석 (Architecture)

현재 v0가 구성한 코드는 **서버(백엔드) 통신이 배제된 순수 프론트엔드(SPA) 구조**입니다. 향후 백엔드 서비스를 도입하거나 제미나이 AI를 활용해 실서버를 고도화할 때, 각각 담당할 영역은 다음과 같습니다.

```mermaid
graph TD
    subgraph Frontend [프론트엔드 (React / Next.js Client)]
        UI[사용자 UI 화면 - Dashboard/Detail]
        State[React Local State - view, selectedId]
        DeviceAPI[센서 측정 상태 시각화]
        VoiceForm[음성 기록/메모 입력 폼]
    end

    subgraph Backend [백엔드 (Next.js API Routes / Express / Gemini API)]
        API[API 서버 - Endpoints]
        DB[(데이터베이스 - PostgreSQL/MongoDB)]
        Gemini[Gemini Flash AI API]
        Kakao[알림톡/SMS 발송 서비스]
    end

    UI -->|1. 사용자 행동 및 음성 입력| API
    API -->|2. 프롬프트 요청| Gemini
    Gemini -->|3. 증상분석 & 임상 리포트 추출| API
    API -->|4. 데이터 저장 및 조회| DB
    API -->|5. 보호자 전송| Kakao
    API -->|6. 동적 JSON 응답| UI
```

### 1) 프론트엔드 영역 (현재 구현 완료된 상태)
* **UI 렌더링**: 사용자(사회복지사)가 입력하는 UI 폼, 차트 컴포넌트, 위험 분류 뱃지 시각화.
* **화면 분기**: 로컬 리액트 상태(`useState`)에 기반한 빠른 화면 전환.
* **센서/차트 시각화**: 기기 연결 상태, 오늘 복약 확인, 7일간 수면 패턴 시계열 차트 렌더링.

### 2) 백엔드 및 AI 영역 (향후 Gemini 연동 시 구축해야 할 상태)
* **데이터 영속성 (DB)**: 하드코딩된 `data.ts` 대신 PostgreSQL, MongoDB 등의 데이터베이스에 어르신 인적 정보와 IoT 센서(약통 개폐 이력, 밴드 수면 정보) 로그를 저장하고 동적으로 조회하는 백엔드 API 필요.
* **Gemini API 연동 (AI 핵심 파트)**:
  * 사회복지사가 앱에서 음성(혹은 메모 텍스트)을 입력하고 분석 요청을 보내면, 백엔드 서버가 이를 수신합니다.
  * 백엔드에서는 **Gemini 1.5/2.0 API**를 호출하여, 입력된 텍스트로부터 다음 항목들을 JSON 구조로 추출하게 만듭니다.
    * 어르신이 겪고 있는 유의미한 증상(Symptom)
    * 복용 약물과의 상관관계 결론(Conclusion)
    * 분석 신뢰도(Confidence)
    * 사회복지사가 취해야 할 실천적 행동 가이드 목록(Care Guide)
    * 의료진용 임상 보고서 요약(Clinical Report)
  * 이를 프론트엔드로 반환하여 화면에 즉시 렌더링시킵니다.
* **실시간 센서 상태 업데이트**: 스마트 기기들(약통, 밴드)의 웹소켓(WebSocket) 또는 IoT MQTT 프로토콜 연동 처리.
* **알림 발송**: 상세 화면의 "카카오톡 알림장 전송하기" 클릭 시, 실제로 보호자의 번호로 SMS나 알림톡이 전송되도록 외부 API(Ncloud Sens, Solapi 등) 연동.

## 5. 제미나이(Gemini)를 통한 향후 수정 및 고도화 계획 제안

토큰 비용을 효율적으로 아끼면서 해커톤의 비즈니스 로직 완성도를 극대화하기 위해 다음과 같이 고도화를 추진할 수 있습니다.

1. **Next.js Route Handlers(API 라우트) 생성**:
   * `app/api/analyze/route.ts` 파일을 신설하여 브라우저에서 직접 Gemini API Key를 노출하지 않고 안전하게 제미나이 모델을 호출하는 백엔드 API를 만듭니다.
2. **실시간 음성 요약 프롬프트 튜닝**:
   * 프론트엔드 상세화면에서 방문자가 기록을 적을 때, 즉시 비동기 통신(`fetch`)으로 API 서버에 전송한 뒤, 실시간 AI 진단 카드가 역동적으로 업데이트되도록 구성합니다.
3. **복약/수면 데이터 임의 조작 데모 모드 추가**:
   * 심사위원들이 수면 차트를 조작하거나 신규 복약 체크를 해제해보고, 그에 따라 위험 점수(`riskScore`)가 실시간으로 재계산되어 트리아지 정렬 순서가 바뀌는 인터랙티브 연출을 추가합니다.
